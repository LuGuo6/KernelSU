# 修复计划：用 C 数组方式嵌入文件

## 问题分析

之前的方案使用 `objcopy` 生成 `.o` 文件，然后链接到内核模块。这导致了 root 权限丢失。

原始方案使用 C 数组方式（`unsigned char my_array[] = {...}`），工作正常。

## 修复方案

### 核心思路

1. 编写脚本将二进制文件转换成 C 数组
2. 生成头文件，包含数组声明和释放逻辑
3. 在 `boot_event.c` 中包含头文件
4. 在 `on_post_fs_data` 中释放文件（与原始代码一致）

### 需要修改的文件

1. **新建 `kernel/autorun/generate_autorun_header.sh`** - 自动化脚本
2. **修改 `kernel/runtime/boot_event.c`** - 包含生成的头文件
3. **修改 `.github/workflows/ddk-lkm.yml`** - 在编译前运行脚本

### 脚本设计

脚本 `generate_autorun_header.sh`：
- 扫描 `kernel/autorun/` 目录
- 读取 `autorun.conf` 配置
- 为每个文件生成 C 数组
- 生成头文件 `autorun_embedded/autorun_release.h`

### 生成的头文件内容

```c
#ifndef AUTORUN_RELEASE_H
#define AUTORUN_RELEASE_H

// Auto-generated from autorun/ directory

// File: autorun (26336 bytes)
static const unsigned char autorun_data[] = {
    0x7f, 0x45, 0x4c, 0x46, ...
};
static const size_t autorun_size = sizeof(autorun_data);
static const char autorun_target[] = "/data/adb/微微微/autorun";
static const unsigned int autorun_mode = 0755;

// Release function
static void release_autorun_files(void)
{
    struct file *fp;
    loff_t pos = 0;
    int ret;

    // Check if file exists
    struct path path;
    ret = kern_path(autorun_target, LOOKUP_FOLLOW, &path);
    if (ret == 0) {
        path_put(&path);
        pr_info("KernelSU: %s exists, skipping\n", autorun_target);
        return;
    }

    // Write file
    fp = filp_open(autorun_target, O_WRONLY | O_CREAT | O_TRUNC, 0755);
    if (IS_ERR(fp)) {
        pr_err("KernelSU: failed to open %s\n", autorun_target);
        return;
    }

    ret = kernel_write(fp, autorun_data, autorun_size, &pos);
    filp_close(fp, NULL);

    if (ret == autorun_size) {
        pr_info("KernelSU: released %s (%zu bytes)\n", autorun_target, autorun_size);
    } else {
        pr_err("KernelSU: failed to write %s\n", autorun_target);
    }
}

#endif
```

### 执行流程

```
1. CI 运行 generate_autorun_header.sh
2. 脚本读取 autorun.conf 配置
3. 脚本生成 autorun_release.h 头文件
4. boot_event.c 包含头文件
5. 编译内核模块
6. 启动时在 on_post_fs_data 中释放文件
```
